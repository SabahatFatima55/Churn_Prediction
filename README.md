# 🛡️ ChurnGuard AI — Customer Churn Prediction & Retention Intelligence System

An end-to-end machine learning project that predicts customer churn for a telecom company using supervised learning, unsupervised clustering, ensemble methods, and hyperparameter tuning — deployed as an interactive Streamlit web application.

## 🎯 Objective

The project applies both supervised and unsupervised machine learning techniques to a real-world telecom dataset in order to predict whether a customer will churn (leave the service) and to surface actionable retention insights. It covers the complete ML pipeline: data preprocessing, feature engineering, model building, evaluation, hyperparameter tuning, and deployment.

---

## 📊 Dataset

- **Source:** [Telco Customer Churn dataset](https://www.kaggle.com/datasets/blastchar/telco-customer-churn) (IBM Sample Dataset)
- **File:** `dataset/WA_Fn-UseC_-Telco-Customer-Churn.csv`
- **Records:** 7,043 customers
- **Churn rate:** 26.5% (imbalanced — handled with SMOTE)
- **Features:** Customer demographics, account information, and subscribed services (gender, tenure, contract type, internet service, payment method, monthly/total charges, etc.)

---

## 🧪 Project Phases

### Phase 1 — Data Preprocessing
- Missing value treatment (`TotalCharges` converted from string, missing values filled for zero-tenure customers)
- Outlier treatment using Winsorization (IQR method)
- Feature engineering — 30+ engineered features including risk scores, loyalty scores, and interaction features
- Encoding (binary mapping + one-hot encoding) and an 80/20 train-test split with a separate validation split for threshold tuning
- Class imbalance handled with SMOTE (applied only to the training set)

### Phase 2 — Model Implementation
**Supervised models (10):** Logistic Regression, Decision Tree, Random Forest, XGBoost, SVM, k-NN, LightGBM, CatBoost, Extra Trees, Gradient Boosting

**Ensemble methods (3):** Stacking Ensemble, Soft Voting Ensemble, Weighted Ensemble (validation-tuned threshold)

**Unsupervised clustering (3):** K-Means, Agglomerative (Hierarchical) Clustering, DBSCAN

### Phase 3 — Evaluation & Visualization
- Classification metrics: Accuracy, Precision, Recall, F1-Score, ROC-AUC
- Clustering metrics: Silhouette Score, Davies-Bouldin Index
- Visualizations: confusion matrices, ROC curves, feature importance plots, cluster profiles, dendrograms

### Phase 4 — Hyperparameter Tuning
- `GridSearchCV` on Random Forest (5-fold CV)
- `RandomizedSearchCV` on LightGBM (5-fold CV, 30 iterations)
- A Tuned Stacking Ensemble built from the optimized base models
- Threshold optimization performed on a held-out validation set to avoid data leakage

### Phase 5 — Deployment
- The best-performing model is exported and served through an interactive Streamlit application (`app.py`)

## 🏆 Results

All models were trained on SMOTE-balanced data and evaluated on an untouched 20% test set.

| Model | Test Accuracy | F1-Score | ROC-AUC |
|---|---|---|---|
| **Voting Ensemble (best overall)** | **0.7850** | 0.6008 | 0.8374 |
| CatBoost (best individual model) | 0.7807 | 0.5940 | **0.8402** |
| Gradient Boosting | 0.7793 | 0.5803 | 0.8360 |
| Tuned Stacking | 0.7779 | 0.5854 | 0.8309 |
| LightGBM | 0.7779 | 0.5765 | 0.8262 |

**Best Model: Voting Ensemble** — a soft-voting combination of LightGBM, CatBoost, and XGBoost (weights = [2, 2, 1]). Ensemble methods outperformed individual models because they combine the strengths of multiple algorithms and average out individual model errors, resulting in more stable generalization on unseen data.

## ⚙️ Setup & Installation

**1. Clone or download the project folder**

**2. Install the required packages**
```bash
pip install pandas numpy scikit-learn xgboost lightgbm catboost imbalanced-learn streamlit matplotlib seaborn scipy
```

**3. Run the notebook**

Open `New_Churn_Prediction.ipynb` in Jupyter or VS Code and run all cells from top to bottom (**Run All**). This executes the full pipeline — preprocessing, model training, evaluation, hyperparameter tuning — and the final cell exports the best model as `churn_model.pkl`.

**4. Launch the Streamlit app**

Make sure `churn_model.pkl` and the `dataset/` folder are in the same directory as `app.py`, then run:
```bash
streamlit run app.py
```

## 🖥️ Application Features

The Streamlit app provides four pages:

- **Home** — Project overview and headline model metrics
- **Predict Churn** — Single-customer prediction with churn probability, risk level, and suggested retention actions
- **Model Performance** — Confusion matrix, feature importance, classification report, and model configuration
- **Batch Prediction** — Upload a CSV of multiple customers and download churn predictions for all of them

## 📌 Notes

- All reported metrics are computed on a held-out test set that the model never saw during training or threshold tuning, ensuring no data leakage.
- Train accuracy is reported on the original (non-SMOTE) training data for an honest, non-inflated performance comparison.
- The deployed model is retrained on the full training set (rather than the smaller sub-split used for model comparison) so that it benefits from all available data before being served in production.
